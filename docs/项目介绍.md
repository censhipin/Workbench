# 表格数据工作台 — 项目介绍

---

## 1. 项目概述

**表格数据工作台**（Table Data Workbench）是一个基于 AI 自然语言处理的桌面端数据处理工具。用户上传 Excel 表格后，可以通过中文自然语言指令完成排序、筛选、求和、去重、多表匹配、合并、数据清洗等操作，无需书写公式或编写代码。

项目形态为 **Electron 桌面应用**，同时兼容浏览器访问模式。所有数据均在本地处理，不上传任何用户文件到外部服务器。

### 3.2 三栏布局

- **左侧栏（Sidebar）**：文件池，负责上传、选择、管理 Excel 文件，宽度固定 240px
- **中间栏（MainPanel）**：核心工作区，包含数据预览、AI 输入框、结果预览三个区域
- **右侧栏（RightPanel）**：任务面板，展示执行计划、结果摘要、操作历史，可折叠

### 3.3 数据处理流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 用户输入  │ →  │ NLU 解析  │ →  │ 执行引擎  │ →  │ 结果展示  │
│ 自然语言  │    │ AI/规则   │    │ 7种操作   │    │ 表格+摘要 │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                      │
                      ↓
                ┌──────────┐
                │ Schema   │
                │ 解析器    │
                │ 语义→列名 │
                └──────────┘
                      │
                      ↓
                ┌──────────┐
                │ 歧义检测  │
                │ 需要确认？│
                ├──是→弹窗  │
                └──否→执行  │
```

### 3.4 NLU 管道（自然语言理解）

用户输入的自然语言指令经过以下管道处理：

1. **分词器**（Tokenizer）：基于词典的最长优先中文分词，提取关键词
2. **意图解析**：优先调用 DeepSeek AI 进行语义理解，失败时降级到规则引擎（关键词匹配）
3. **Schema 解析器**：将语义目标（如"销售额""工资"）映射到实际表格列名，支持同义词模糊匹配
4. **歧义检测**：当匹配到多个候选列时，弹出确认框让用户选择
5. **任务意图**：最终生成标准化的 TaskIntent 对象，交给执行引擎

### 3.5 执行引擎

执行引擎负责接收标准化的指令并操作数据，支持 7 种操作：

| 操作 | 说明 | 多列支持 | 分组支持 |
|------|------|----------|----------|
| 排序（sort） | 按指定列升序或降序排列 | × | × |
| 筛选（filter） | 按条件过滤数据行（等于、包含、日期范围） | × | × |
| 求和/聚合（sum） | 单列或多列数值求和，支持 AVG/COUNT/MAX/MIN | ✅ | ✅ |
| 去重（dedup） | 按关键列去除重复行 | × | × |
| 多表匹配（match） | 多个表按关键字段关联（类似 VLOOKUP） | × | × |
| 多表合并（merge） | 多个表纵向拼接 | × | × |
| 数据清洗（clean） | 移除空行、修复异常单元格值 | × | × |

执行完成后会经过**结果验证器**（ResultVerifier）检查数据正确性，然后生成**执行报告**（PlanStep）展示给用户。

## 4. 数据结构

### 4.1 核心类型

- **WorkbenchFile**：文件实体，包含多张 Sheet（SheetInfo），每张 Sheet 包含列定义（ColumnDef）和数据行（RowData）
- **TaskIntent**：用户意图的结构化表示，包含操作类型、目标列、分组条件、聚合方式、筛选条件等
- **ColumnDef**：列定义，包含 key（字段标识）、title（中文列名）、type（text/number/date）
- **RowData**：数据行，键值对结构 `Record<string, string | number | null>`

### 4.2 语义概念注册表

系统内置了一套语义概念到列名关键词的映射表，用于 NLU 的 Schema 解析：

| 概念 | 匹配的列名关键词 |
|------|------------------|
| 销售额 | 金额、销售金额、销售额、总金额、收入、营收、营业额 |
| 工资 | 工资、薪资、薪酬、基本工资、底薪 |
| 绩效 | 绩效、奖金、绩效奖金、绩效工资、提成 |
| 加班费 | 加班、加班补贴、加班费、加班工资 |
| 扣除 | 扣除、扣款、扣除项、扣减、罚款 |
| 姓名 | 姓名、名字、员工姓名 |
| 手机号 | 手机、手机号、电话、联系电话 |
| 部门 | 部门、科室、组、团队、事业部 |

## 5. 数据存储

| 存储内容 | 存储位置 | 技术方案 |
|----------|----------|----------|
| 上传的文件数据 | 浏览器 IndexedDB | files 对象存储 |
| 操作历史记录 | 浏览器 IndexedDB | meta 对象存储 |
| 任务文件勾选状态 | 浏览器 IndexedDB | meta 对象存储 |
| DeepSeek API Key | 浏览器 localStorage | 明文存储 |
| DeepSeek API Key（环境变量） | .env.local 文件 | 开发环境回退方案 |

所有数据均在本地处理，不上传任何用户文件到外部服务器。仅有自然语言指令的解析请求会发送到 DeepSeek API（需用户配置 API Key）。

## 6. 质量检查模块

系统内置数据质量检查功能（Data Audit），可对当前表格进行全方位体检：

- **数据统计**：总行数、总列数、空白单元格数、空白行数
- **重复检测**：基于全文签名的重复行识别
- **空值检测**：关键字段（姓名、手机号、邮箱、身份证等）的空值定位
- **异常检测**：
  - 数值异常（超出合理范围的值）
  - 邮箱格式校验
  - 手机号格式校验（中国大陆 11 位手机号）
  - 身份证号格式校验（15/18 位）
- **质量评分**：综合打分并给出等级（A/B/C/D）
- **修复建议**：针对每项问题提供自动修复方案

---

## 7. 质量引擎（Quality Engine）

在 Data Audit 基础上，新增了更细粒度的质量检测引擎：

- **规则检测**：通用数据质量规则（空值率、格式正确率等）
- **列类型推断**：根据列数据自动推断类型（姓名、手机号、邮箱、身份证、日期等）
- **业务规则**：可配置的领域特定业务规则
