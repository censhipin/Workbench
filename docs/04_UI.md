# UI Architecture

## 布局结构（5 栏）

```
TopBar (56px) — logo | 文件名—Sheet名 | version | [设置] [帮助]
├── LeftPanel (260px)
│   ├── Workspace — 文件列表（4个 mock + 用户上传）
│   └── WorkflowTree — 操作步骤树（版本树形视图）
├── MainPanel (flex:1)
│   ├── DataTabs — [📄原始数据] [📊执行结果] [📉对比]
│   ├── StatusBar — 当前状态文本
│   ├── DataArea — DataPreview | ResultPreview | CompareView
│   └── BottomBar — 输入框 + 执行按钮 + 快捷操作
└── RightPanel (320px)
    ├── PlanViewToggle — [人类可读] [开发者]
    ├── ExecutionPlan — 5步执行计划展示
    └── VersionTimeline — 版本时间线（按版本号降序）
```

## 组件树

```
page.tsx (所有状态中心)
  ├── TopBar — 应用标题、文件-表名、版本标签、设置/帮助按钮
  ├── LeftPanel
  │     ├── Workspace — 文件卡片（文件图标、名称、行数列数、Sheet选择、删除、任务勾选）
  │     └── WorkflowTree — 版本树（按 parentVersion 构建分支树）
  ├── MainPanel
  │     ├── DataTabs — tab 切换
  │     ├── StatusBar — 提示文字
  │     ├── DataPreview — 原始数据表格（含编辑模式、拖拽排序、审计按钮）
  │     ├── ResultPreview — 执行结果表格（含全屏查看、导出）
  │     ├── CompareView — 左右对比视图
  │     └── BottomBar — 输入 + 提交 + 快捷操作
  ├── RightPanel
  │     ├── ExecutionPlan — 执行步骤卡片
  │     └── VersionTimeline — 版本节点时间线（含删除、设为当前、全屏查看）
  ├── OperationHistoryModal — 操作历史弹窗
  ├── ConfirmationDialog — 歧义确认弹窗
  └── DataAudit — 数据审计面板
```

## 状态管理

所有状态集中在 `page.tsx`，通过 React hooks 管理：

| 状态 | 类型 | 用途 |
|------|------|------|
| `files` | WorkbenchFile[] | 文件列表 |
| `selectedFileId` | string | 选中的文件 |
| `activeSheet` | string | 当前工作表 |
| `taskFileIds` | string[] | 勾选的任务文件 |
| `promptText` | string | 输入框内容 |
| `isRunning` | boolean | 执行中标志 |
| `executionSteps` | PlanStep[] | 5 步执行计划 |
| `error` | string | 错误信息 |
| `versions` | Version[] | 版本列表 |
| `currentVersionId` | string | 当前显示版本 |
| `baseVersionId` | string | 执行基准版本 |
| `activeTab` | DataTab | 当前 tab |
| `historyItems` | HistoryItem[] | 操作历史 |

## 交互流程

```
1. 用户选择/上传文件 → Workspace → setSelectedFileId
2. 输入自然语言指令 → BottomBar → setPromptText
3. 点击执行 → handleSubmit()
   ├── AI 理解或规则解析
   ├── 歧义检测 (AmbiguityDetector)
   │   ├── 有歧义 → ConfirmationDialog 等待用户确认
   │   └── 无歧义 → 继续执行
   └── executeIntent()
4. 执行完成 → createVersion() → setVersions + setCurrentVersionId
5. 结果展示 → ResultPreview → DataTable
6. 版本列表 → VersionTimeline 更新
7. 执行历史 → historyItems 追加
```
