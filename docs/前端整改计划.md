## 各区域详细说明

### 1. Top Bar — 顶部导航栏

从左到右：
- 产品名称：「数据工作台」
- 当前文件名（如：客户.xlsx）
- 当前版本号（如：版本 v5）
- 右侧：[设置] [帮助] 图标或文字按钮

---

### 2. 左栏上 — Workspace（工作区）

固定宽度约 220px，固定高度，内容超出纵向滚动。

- 标题：「📁 Workspace」
- 文件列表：上传的 Excel 文件按列表排列，当前打开的文件高亮
- 每行：文件名 + 选中状态
- 底部：「上传文件」按钮
- 点击文件 → 加载该文件到预览区（原始数据 Tab），版本历史重置
- 文件可删除。删除时只移除引用，版本数据保留在后台，操作历史仍可查看

---

### 3. 左栏下 — Workflow（操作流程树）

与 Workspace 均分左栏高度。

- 标题：「🌲 Workflow」
- 树形结构展示所有操作步骤的依赖关系

```
原始
├ v1 筛选
│ ├ v2 排序
│ │ └ v3 新增奖金
│ └ v4 匹配工资
└ v5 新增奖金
```
- 缩进表示「基于上一步」的依赖关系
- 当前步骤高亮
- 点击任意步骤 → 预览区的「执行结果」Tab 切换到该版本数据
- 纯文本节点，不展示数据内容

左下角还一个按钮：「📜 操作历史」，点击弹出全局操作历史（见后文）。

---

### 4. 中间 — 结果预览区（主工作区）

占据页面的主要宽度，剩余宽度自动填满。

#### Tab 栏（预览区顶部）

| Tab | 说明 |
|-----|------|
| 📄 原始数据 | 始终显示刚上传时的原始表格，永远不会变 |
| 📊 执行结果 | 显示当前选中的版本数据，随执行更新 |
| 📉 对比 | 原始数据和执行结果左右并排显示 |

#### Tab 交互规则

- **刚上传文件、未执行任务**：「原始数据」选中亮色，「执行结果」和「对比」灰色不可点
- **执行过一次后**：自动切换到「执行结果」Tab 显示最新结果，「对比」同时亮起可用
- **点击「原始数据」Tab**：显示原始表，顶部提示条：「当前查看原始数据（非结果数据）」
- **点击「对比」Tab**：左右并排显示原始表和当前结果

#### 对比视图样式

```
┌──────────────────────┬──────────────────────┐
│ 原始数据 (50行×8列)  │ v5 结果 (20行×9列)   │
├──────────────────────┼──────────────────────┤
│ 姓名 │ 部门 │ 工资   │ 姓名 │ 部门│工资│奖金│
│ 张三 │ 技术 │ 8000   │ 张三 │ 技术│8000│ 高 │
└──────────────────────┴──────────────────────┘
```

### 5. 右栏上 — AI 执行计划

固定宽度约 240px。

**默认模式：人类可读**

```
Step 1: 筛选部门为技术部 ✓
Step 2: 按工资降序排序 ✓
Step 3: 新增奖金列
         如果工资 > 5000 则 "高"，否则 "低" ✓
```

**右上角「<> 开发者模式」切换**：点击切换为显示当前步骤的完整 JSON Plan（等宽字体）。

只显示当前版本对应的执行计划，不堆叠历史。

### 6. 右栏下 — 版本时间线

与 AI 计划均分右栏高度。

**标题：「📊 版本时间线」**

**纵向时间线展示，带连接线：**

```
┌─────────────────────┐
│  ● v5 ✅ 当前       │
│    20行×9列         │
│    新增奖金         │
├─────────────────────┤
│  ● v4              │
│    20行×8列         │
│    匹配工资         │
├─────────────────────┤
│  ● v3              │
│    20行×8列         │
│    筛选             │
└─────────────────────┘
```

**交互：**
- 点击任意版本节点 → 预览区「执行结果」Tab 切换到该版本数据
- 点击节点右上角「···」弹出操作菜单：
  - 「📌 设为当前」— 将此版本设为主工作区数据，后续操作基于此
  - 「👁 完整查看」— 浮层全屏展示该版本完整数据
  - 「✕ 删除此版本」— 删除该版本及所有后续版本，确认后执行

### 7. 底部操作栏

贯穿页面底部，固定高度。

**左侧：当前上下文信息**

```
当前：v5 · 来源：客户.xlsx · 20行×9列
```

**中间：输入框**

- 占主要宽度
- placeholder：「请输入指令，如：筛选部门为技术部的数据」
- 回车或点击右侧「执行」提交

**右侧：操作按钮**

| 按钮 | 功能 |
|------|------|
| 执行 | 提交当前输入，生成新版本并追加到时间线 |
| ↩ 撤销 | 回退一步（删除最新版本） |

**输入框下方文字链接：**
- 「📌 重置到原始数据」— 清空所有版本历史，回到原始数据

---

### 8. 操作历史（全局）

通过左下角「📜 操作历史」按钮触发，弹出模态窗口。

**展示：** 全局流水账，无论当前打开什么文件，显示所有文件的所有操作记录。

```
[10:32] 客户.xlsx → 筛选部门=技术部 → v1
[10:35] 客户.xlsx → 按工资排序 → v2
[10:38] 客户.xlsx → 新增奖金列 → v3
[10:42] 工资.xlsx → 上传文件
[10:45] 工资.xlsx → 统计总和 → v1
```

**交互：**
- 点击任意历史记录 → 浮层中展示该版本数据
- 浮层底部按钮：「在当前工作区打开」
  - 如果源文件还在 Workspace 中 → 切换到该文件 + 该版本
  - 如果源文件已被删除 → 按钮变灰，提示「源文件已移除，无法继续操作」（但数据仍可查看）

## 核心数据状态

### 版本数据结构

```typescript
interface Version {
  id: string;            // uuid
  fileId: string;        // 所属文件
  version: number;       // v1, v2, v3...
  parentVersion?: string; // 上一个版本的 id
  operation: string;     // 操作摘要（"筛选部门=技术部"）
  plan: object;          // 完整的 ExecutionPlan JSON
  columns: ColumnDef[];
  rows: RowData[];
  createdAt: string;
}
```

### 用户操作流程

```
上传文件 → 看到原始数据（「原始数据」Tab）
  → 输入指令 → 执行 → 结果自动追加到版本时间线
  → 预览区切换到「执行结果」Tab
  → 再输入指令 → 基于上一步结果继续执行
  → 点击版本时间线任意节点 → 预览区切换数据
  → 点击「设为当前」→ 后续操作基于该版本
  → 点击「重置到原始数据」→ 清空所有版本
```

### 后端接口需求（前端整改可能需要）

| 接口 | 说明 |
|------|------|
| POST /execute | 提交指令 + 当前版本信息，返回新版本数据 |
| GET /files | 获取 Workspace 文件列表 |
| GET /versions?fileId=X | 获取某文件的所有版本 |
| DELETE /versions/:id | 删除某版本及后续版本 |
| GET /history | 获取全局操作历史 |
| DELETE /files/:id | 删除文件（仅移除引用，版本数据保留） |

## 关于后端记忆状态

每次执行后，当前 columns + rows 作为下一次执行的 mainSheet。后端不需要额外状态管理，执行引擎本来就是 `(plan, context) → result` 的纯函数 —— 前端把上一步结果的 columns + rows 传回给后端即可。
