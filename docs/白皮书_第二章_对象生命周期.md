# 第二章：对象生命周期 (Object Lifecycle)

## 2.1 TaskPlan

| 属性 | 值 |
|------|-----|
| 谁创建 | `deepseek.ts` parseResponse() — AI 路径；`rule-taskplan-converter.ts` ruleIntentToTaskPlan() — 规则路径 |
| 谁修改 | `field-resolver.ts` resolveTaskPlan() — 修正 columnHint 为精确列名（创建副本，不修改原对象） |
| 谁消费 | `task-compiler.ts` compile() → 编译为 ExecutionPlan |
| 生命周期 | 短暂存在：创建 → 编译 → GC |
| 是否持久化 | 否 |
| 是否可序列化 | 是（JSON） |

## 2.2 TaskIntent

| 属性 | 值 |
|------|-----|
| 谁创建 | `taskplan-converter.ts` taskPlanToIntent() — AI 路径；`semantic-parser.ts` parse() — 规则路径 |
| 谁修改 | page.tsx handleConfirmAmbiguity() 设置 `resolvedColumns`；index.ts 设置 `v2plan` |
| 谁消费 | `execution-engine.ts` runExecutionEngine() 检查 intent.v2plan；PlanStepBuilder.build() 读取展示信息；page.tsx AmbiguityDetector.detect() 检查歧义 |
| 生命周期 | 短暂存在：创建 → 附加 v2plan → 消费 → GC |
| 是否持久化 | 否 |
| 是否可序列化 | 是 |

## 2.3 ExecutionPlan（含子类型：FilterPlan / SortPlan / AggregatePlan / ...）

| 属性 | 值 |
|------|-----|
| 谁创建 | `task-compiler.ts` compile() — 每种 action 对应一个 compileXxx() 函数 |
| 谁修改 | `plan-validator.ts` validatePlan() — 返回标准化副本（filter/update 做类型转换）；`execution-engine.ts` PipelineExecutor 读取 `.output` 约束 |
| 谁消费 | `execution-engine.ts` runExecutionPlan() — 调度到 ExecutorRegistry；各 Executor.execute() 读取参数；`verifier/` 各 Verifier.verify() 读取做校验 |
| 生命周期 | 短暂存在：编译 → 校验 → 执行 → GC |
| 是否持久化 | 否 |
| 是否可序列化 | 是 |

### 2.3.1 各子类型字段表

| Plan 类型 | 关键字段 | 创建位置 |
|-----------|---------|---------|
| FilterPlan | conditions: ConditionExpr[] | compileFilter() |
| SortPlan | sorts: SortClause[] | compileSort() |
| AggregatePlan | method, columns, groupBy | compileAggregate() |
| DedupPlan | columns | compileDedup() |
| MatchPlan | matchColumns, lookupTables | compileMatch() |
| MergePlan | sourceTables | compileMerge() |
| CleanPlan | (无参) | compile() switch case 'clean' |
| ProjectionPlan | includeColumns/excludeColumns/renameColumns/reorderColumns | compileProjection*() |
| UpdatePlan | column, value, conditions | compileUpdate() |
| FormulaPlan | targetColumn, sourceColumns, expressionType, ... | compileFormula() |
| PipelinePlan | steps: ExecutionPlan[] | compilePipeline() |

## 2.4 Dataset（{ columns: ColumnDef[], rows: RowData[] }）

| 属性 | 值 |
|------|-----|
| 谁创建 | `file-engine.ts` parseFile() — 上传文件时；`data-engine.ts` 各函数 — 执行时；page.tsx `activeDataset` — 由 currentSheet 变化自动设置 |
| 谁修改 | **没有任何地方修改** — 全部替换为新对象 |
| 谁消费 | handleSubmit 读取 activeDataset；executor.execute 读取做计算；DataPreview/ResultPreview 读取做展示 |
| 生命周期 | **持久化** — 文件 via IndexedDB（loadFiles/saveFile），数据源贯穿整个 session |
| 是否持久化 | 文件数据 → IndexedDB 持久化；版本数据 → 仅内存 |

### 2.4.1 Dataset 引用关系（关键）

```
activeDataset (page.tsx state)
    │ 指向 currentSheet 的引用（不是克隆）
    │ 但 executeIntent 时被 JSON.parse(JSON.stringify) 深度克隆
    ▼
executor 拿到的是克隆，不引用 UI state
    │
    ▼
结果被深度克隆后存入 Version
    │
    ▼
Version 数据只用于展示，不参与下一次执行
```

**关键规则：**
- `activeDataset` 不是克隆——它直接引用 `currentSheet.columns/rows`
- 但每次 `executeIntent` 执行前做 `JSON.parse(JSON.stringify())` 深度克隆
- 结果存入 `Version` 时也是克隆
- `Version` 仅用于显示，不参与执行
- 所以**永远不要**修改 `Version` 中的数据作为下一次执行的输入

## 2.5 Version

| 属性 | 值 |
|------|-----|
| 谁创建 | page.tsx `createVersion()` — 每次 executeIntent 成功后 |
| 谁修改 | **不可变** — 创建后不修改 |
| 谁消费 | ResultPreview、CompareView、VersionTimeline 做展示；handleUndo/handleReset 做移除 |
| 生命周期 | 内存中，最多 20 个版本，从 session 开始到结束 |
| 是否持久化 | 否 |
| 是否可序列化 | 是（但不会存到 IndexedDB） |

## 2.6 HistoryItem

| 属性 | 值 |
|------|-----|
| 谁创建 | page.tsx executeIntent 成功分支 |
| 谁修改 | **不可变** |
| 谁消费 | OperationHistoryModal UI 展示；handleHistoryClick 从历史恢复版本 |
| 生命周期 | 持久化到 IndexedDB，最多 50 条 |
| 是否持久化 | 是（saveHistory/loadHistory） |

## 2.7 EngineRunResult

| 属性 | 值 |
|------|-----|
| 谁创建 | `execution-engine.ts` runExecutionEngine() |
| 谁修改 | 不修改（page.tsx 从其中提取数据） |
| 谁消费 | page.tsx executeIntent() — 读取 success/resultData/error |
| 生命周期 | 短暂存在：返回 → 消费 → GC |
| 是否持久化 | 否 |

## 2.8 PlanStep[]

| 属性 | 值 |
|------|-----|
| 谁创建 | `execution-engine.ts` PlanStepBuilder.build() |
| 谁修改 | page.tsx — 动画循环：waiting → executing → completed/failed（修改 status） |
| 谁消费 | ExecutionPlan 组件 UI 展示 |
| 生命周期 | 短暂存在：每次执行重新创建 |
| 是否持久化 | 否 |

## 2.9 对象间关系图

```
用户输入 prompt
    │
    ▼
TaskPlan (AI 输出 JSON / 规则路径转换)
    │
    ├──→ TaskIntent (转换，附加 v2plan)
    │       │
    │       ▼
    │   ExecutionPlan (编译产生)
    │       │
    │       ▼
    │   EngineRunResult (执行产生)
    │       │
    │       ├──→ Version (成功时创建，持久化到 state)
    │       ├──→ HistoryItem (成功时创建，持久化到 IndexedDB)
    │       └──→ PlanStep[] (用于 UI 动画)
    │
    └──→ SchemaResolution (列名解析结果，用于歧义检测)
```

## 2.10 对象可变性总表

| 对象 | 创建后是否可修改 | 修改者 |
|------|----------------|--------|
| TaskPlan | 否 (替换) | field-resolver 创建副本修改后返回 |
| TaskIntent | 是 | page.tsx 设置 resolvedColumns, index.ts 设置 v2plan |
| ExecutionPlan | 否 (替换) | validatePlan 创建标准化副本 |
| ConditionExpr | 否 | 创建即固定 |
| FilterCondition | 否 | 创建即固定 |
| EngineRunResult | 否 | 创建即固定 |
| ExecutionResult | 否 | 创建即固定 |
| ExecutorResult | 否 | 执行器返回 |
| Version | 否 (不可变) | 只读 |
| HistoryItem | 否 (不可变) | 只读 |
| PlanStep[] | 是 (内部) | page.tsx 动画循环修改 status |
| WorkbenchFile | 否 (替换) | 上传时创建，删除时移除 |
| SheetInfo | 否 (替换) | 无修改操作 |
| ColumnDef | 否 | OutputProcessor rename 创建新对象 |
| RowData | 否 | 执行器创建新数组，不修改输入 |
| AuditReport | 否 | 创建即固定 |
| Dataset (active) | 否 | 完全替换 |
