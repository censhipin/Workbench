# 第一章：数据流 (Data Flow)

## 1.1 用户输入 → 执行结果（完整链路）

```
用户输入 prompt
    │
    ▼
handleSubmit()                          [app/page.tsx:259]
    │ 读取: promptText, activeDataset, selectedFile, taskFileIds
    │ 输出: calls executeIntent(intent, mainFile, sheetName, taskFileIds)
    │ 错误: setError('...'), 不抛出异常
    │ 副作用: setIsRunning(true), setError(null)
    │
    ▼
parseIntentWithAI(prompt, fileName, columns, rows, fileNames)  [lib/nlu/index.ts:52]
    │ 输入: prompt: string, columns: ColumnDef[], rows: RowData[]
    │ 输出: { intent: TaskIntent|null, resolution: SchemaResolution, aiUsed: boolean }
    │ 副作用: 修改全局 PipelineTrace 单例
    │ 错误1(quota): 返回 intent: null, aiUsed: false
    │ 错误2(AI失败): 降级到规则解析器
    │ 错误3(仍失败): 返回 intent: null（由 handleSubmit 捕获）
    │
    ├── [AI 路径]
    │   deepseekUnderstand()                    [lib/nlu/deepseek.ts:289]
    │   │ 输出: TaskPlan (新对象)
    │   │
    │   taskPlanToIntent(plan)                  [lib/nlu/taskplan-converter.ts:68]
    │   │ 输出: TaskIntent (新对象)
    │   │ 不做: 列解析、数据操作
    │   │
    │   compile(plan, columns, rows)            [lib/v2/task-compiler.ts:50]
    │   │ 输入: TaskPlan (columnHints=中文名称)
    │   │ 输出: CompileResult { success, plan?: ExecutionPlan, error? }
    │   │ 不修改: 输入数据
    │   │ 错误: 列名找不到时返回 error
    │   │
    │   intent.v2plan = compiled.plan
    │
    └── [规则解析路径]
        RuleBasedSemanticParser.parse()         [lib/nlu/semantic-parser.ts:88]
        │ 输出: TaskIntent (新对象)
        │
        ruleIntentToTaskPlan(intent)            [lib/nlu/rule-taskplan-converter.ts:37]
        │ 输出: TaskPlan (新对象)
        │
        compile(plan, columns, rows)            [lib/v2/task-compiler.ts:50]
            intent.v2plan = compiled.plan
    │
    ▼
executeIntent(intent, mainFile, sheetName, taskFileIds)  [app/page.tsx:348]
    │ 输入: intent (已有 v2plan 字段)
    │ 不做: 修改 intent
    │
    ▼
JSON.parse(JSON.stringify(activeDataset))    [app/page.tsx:367-368]
    │ 深度克隆输入数据（切断与 UI 的所有引用）
    │
    ▼
runExecutionEngine(intent, execFile, sheetName, taskFiles)  [lib/execution-engine.ts:196]
    │
    ├── 前置校验
    │   检查: intent, intent.operation, intent.v2plan, sheet 必须存在
    │   输出: ValidationResult { valid, issues }
    │   错误: issues 有 error → 不执行, 直接返回 EngineRunResult { success: false }
    │
    ├── buildTaskSheets()
    │   输出: { columns, rows, name }[]
    │   数据来源: taskFiles[0]是主文件, 其余为辅助表
    │
    ▼
runExecutionPlan(v2plan, currentSheet, taskSheets)  [lib/v2/execution-engine.ts:60]
    │
    ├── validatePlan(plan, columns)            [lib/v2/plan-validator.ts]
    │   纯函数, 不修改输入
    │   输出: { valid, plan(标准化副本), issues }
    │   错误: valid=false → 提前返回 ExecutionResult { success: false }
    │
    ├── createSnapshot(columns, rows)          [lib/v2/execution-snapshot.ts]
    │   JSON.parse(JSON.stringify(...)) 深度克隆
    │   输出: 新的 { columns, rows }（与输入无引用）
    │
    ├── registry.get(plan.type)                [lib/v2/executors/registry.ts]
    │   输出: OperationExecutor | undefined
    │   错误: undefined → 返回 "不支持的操作"
    │
    ├── executor.execute(plan, context)        [各 Executor]
    │   输入: ExecutionContext { mainSheet, taskSheets, output }
    │   输出: ExecutorResult { result: { columns, rows }, summary }
    │   不修改: 输入数据

    │
    ├── runOutputProcessor(rows, columns, output)  [lib/v2/output-processor/OutputProcessor.ts]
    │   处理: includeColumns → excludeColumns → renameColumns → reorderColumns → limit
    │   输出: { columns, rows } (新对象)
    │   不修改: 输入
    │
    ├── runVerification(plan, input, inputRows, outputRows)  [lib/v2/verifier/]
    │   输出: VerificationResult { passed, checks }
    │   错误: passed=false → 返回 ExecutionResult { success: false }
    │
    ├── cloneResult(processed)                 [lib/v2/execution-snapshot.ts]
    │   最终深度克隆（确保输出不引用输入）
    │
    └── return ExecutionResult { success, data, summary, confidence, error? }
    │
    ▼
PlanStepBuilder.build(intent, validation, execution, verification)  [lib/execution-engine.ts:103]
    │ 输出: PlanStep[]（5步：解析意图→验证参数→执行→验证结果→生成报告）
    │ 纯函数, 仅用于 UI 展示
    │
    ▼
setExecutionSteps(steps)                    [app/page.tsx:406]
    │ 动画循环: waiting → executing → completed/failed
    │
    ├── [成功]
    │   createVersion(promptText, intent, resultColumns, resultRows)
    │   → 追加到 versions[] state（最多20个）
    │   → setCurrentVersionId, setActiveTab('result')
    │   → saveHistory() 到 IndexedDB
    │
    └── [失败]
        setError(engineResult.error)
        → errorDialog 自动显示
```

## 1.2 每个节点的四问

| 节点 | 输入 | 输出 | 修改数据？ | 失败返回 |
|------|------|------|-----------|---------|
| handleSubmit | state: promptText, activeDataset, taskFileIds | 调用 executeIntent | 否 | setError + 停止执行 |
| parseIntentWithAI | prompt, columns, rows, fileNames | { intent, resolution, aiUsed } | 仅修改 trace 单例 | intent=null, aiUsed=false |
| deepseekUnderstand | prompt, columns, rows | { success, plan } | 否 | success=false |
| taskPlanToIntent | TaskPlan | TaskIntent | 否 | N/A |
| ruleIntentToTaskPlan | TaskIntent | TaskPlan | 否 | N/A |
| compile | TaskPlan, columns | CompileResult | 否 | success=false + error |
| executeIntent | intent, mainFile, sheetName | 修改 state | 否 | setError |
| runExecutionEngine | intent, file, taskFiles | EngineRunResult | 否 | success=false + error |
| runExecutionPlan | ExecutionPlan, sheets | ExecutionResult | 否 | success=false + error |
| validatePlan | plan, columns | { valid, plan } | 否(返回标准化副本) | valid=false |
| executor.execute | plan, context | ExecutorResult | 否 | throw |
| runOutputProcessor | rows, columns, output | { columns, rows } | 否 | 返回输入 |
| runVerification | plan, input, output | VerificationResult | 否 | passed=false |
| PlanStepBuilder | intent, validation, exec, verify | PlanStep[] | 否 | N/A |

## 1.3 关键数据在哪里被创建

| 对象 | 创建位置 | 备注 |
|------|---------|------|
| TaskPlan | deepseek.ts `parseResponse` 或 rule-taskplan-converter.ts | AI路径创建/规则路径转换 |
| TaskIntent | taskplan-converter.ts `taskPlanToIntent` 或 semantic-parser.ts | AI路径转换/规则路径直接产生 |
| ExecutionPlan | task-compiler.ts `compile()` | 所有路径必经 |
| EngineRunResult | execution-engine.ts `runExecutionEngine()` | 唯一执行结果格式 |
| ExecutionResult | execution-engine.ts `runExecutionPlan()` | V2 引擎输出 |
| Version | page.tsx `createVersion()` | 每个成功执行产生一个 |
| HistoryItem | page.tsx executeIntent 成功分支 | 每个成功执行产生一个 |
| PlanStep[] | execution-engine.ts `PlanStepBuilder.build()` | 每次执行重建 |

## 1.4 rows 被谁读写

| 位置 | 读 | 写 | 修改? |
|------|----|----|--------|
| activeDataset (page.tsx) | handleSubmit 读作输入 | 由 currentSheet 变化自动设置 | 否 |
| snapRows (page.tsx:368) | JSON.parse(JSON.stringify) | - | 否 |
| mainSheet.rows (task-compiler) | compile 传入 ctx | 不修改 | 否 |
| createSnapshot | 读入做深度克隆 | 输出新数组 | 否 |
| executor.execute | 读入做计算 | 输出新 rows | 不修改输入 |
| runOutputProcessor | 读入做列筛选/限制 | 输出新数组 | 否 |
| cloneResult | 读入做最终克隆 | 输出新数组 | 否 |
| createVersion | 接收结果数据 | 存储到 state | 否(存的是克隆) |
| displayRows (page.tsx:174) | 当前版本或原始数据 | 不修改 | 否 |

## 1.5 columns 被谁读写

| 位置 | 读 | 写 | 修改? |
|------|----|----|--------|
| compile() | 读 columns 做列名解析 | 不修改 | 否 |
| validatePlan() | 读 columns 做校验 | 不修改(标准化副本是 plan) | 否 |
| executor.execute() | 读 columns 做计算 | 输出新 columns | 否(新增列时创建新数组) |
| runOutputProcessor() | 读 columns 做筛选/重命名 | 输出新数组 | 否 |
| createSnapshot | 深度克隆 | 输出新数组 | 否 |
| createVersion | 接收结果 | 存储到 state | 否 |

## 1.6 完整数据转换链（以筛选为例）

```
用户: "筛选销售额大于10000"
    │
    ▼
prompt: string
    │
    ▼ (deepseekUnderstand / RuleBasedSemanticParser)
TaskPlan { action: 'filter', conditions: [{ columnHint: '销售额', operator: '>', value: '10000' }] }
    │
    ▼ (taskPlanToIntent / ruleIntentToTaskPlan)
TaskIntent { operation: 'filter', target: '销售额', filters: [{ column: '销售额', operator: 'gt', value: '10000' }] }
    │
    ▼ (compile)
ExecutionPlan → FilterPlan { type: 'filter', conditions: [{ columnKey: 'salesAmount', operator: Operator.GT, value: 10000 }] }
    │
    ▼ (validatePlan)
FilterPlan (已验证/标准化)
    │
    ▼ (FilterExecutor.execute → filterRowsMulti → evaluateCondition)
ExecutorResult { result: { columns, rows }, summary }
    │
    ▼ (runOutputProcessor / cloneResult)
{ columns, rows } (应用了 OutputSpec 限制 + 深度克隆)
    │
    ▼
EngineRunResult → page.tsx → Version → UI
```
