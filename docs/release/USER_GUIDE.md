## 一、快速入门

### 1. 上传 Excel 文件

**操作步骤：**

1. 打开应用，左侧面板为空状态
2. 点击左上角的 **"上传"** 按钮
3. 选择一个 `.xlsx` / `.xls` / `.csv` 文件
4. 文件将显示在左侧文件列表中

> 提示：支持同时上传多个文件用于后续的匹配/合并操作。

### 2. 执行第一条指令

```
1. 在底部输入框输入：
   "帮我计算每个部门平均工资"

2. 系统执行流程：
   ┌─ ① 分析：识别意图 → 按部门AVG计算基本工资
   ├─ ② 修复：自动修正列引用
   ├─ ③ 执行：分组计算平均工资
   ├─ ④ 验证：检查结果正确性
   └─ ⑤ 解释：生成执行说明

3. 结果展示在右侧面板：
   - 数据表格 ✓
   - 执行步骤 ✓
   - 执行说明 ✓
```

### 3. 查看结果

| 区域 | 内容 |
|------|------|
| 中间主面板 | 执行结果数据表格 |
| 右侧执行计划 | 6 步执行过程卡片 |
| 右侧解释面板 | 操作汇总 + 数据说明 |
| 版本时间线 | 历史操作快照列表 |

## 二、典型使用场景

### 场景 1：数据筛选

> "筛选技术部的员工"

**系统理解：**
- 操作：筛选 (Filter)
- 条件：部门 = 技术部

**结果：** 仅保留技术部员工行。

**更多示例：**
```
"工资大于8000的"
"筛选2024年1月的数据"
"找杭州的客户"
"累计消费大于5000的VIP客户"
```

### 场景 2：数据清洗

> "删除空值的行"

**系统理解：**
- 操作：清理 (Clean)
- 行为：移除包含空值的行

**更多示例：**
```
"清理手机号为空的"
"去掉空格"
```

### 场景 3：公式计算

> "新增一列收入，收入 = 基本工资 × 绩效奖金"

**系统理解：**
- 操作：公式 (Formula)
- 目标列：收入
- 公式：基本工资 × 绩效奖金

**结果：** 每行新增"收入"列，存储计算结果。

**更多示例：**
```
"金额 = 数量 × 单价"
"利润 = 小计 - 成本"
"税后工资 = 基本工资 * 0.9"
```

### 场景 4：分组聚合

> "按部门统计工资总和"

**系统理解：**
- 操作：聚合 (Aggregate)
- 方法：SUM（求和）
- 分组：部门
- 目标：基本工资

**结果：** 每个部门一行，显示工资总和。

**更多示例：**
```
"计算各城市平均消费"
"统计每月的订单数量"
"找每个部门的最高工资"
```

### 场景 5：多表匹配

> "把城市信息匹配到客户表"

**系统理解：**
- 操作：匹配 (Match)
- 左表：客户表（当前文件）
- 右表：包含城市信息的表
- 匹配键：自动检测共有列

**结果：** 左表所有行保留，匹配到的右表值以 `_lkp_` 前缀填充。

### 场景 6：流水线操作

> "先筛选技术部，再按工资从高到低排序"

**系统理解：**
- 操作：流水线 (Pipeline)
- 步骤 1：筛选技术部
- 步骤 2：按工资降序排序

**结果：** 两步连续执行，显示最终结果。

**更多示例：**
```
"筛选工资大于10000的，然后计算平均工资"
"删除空值，再按部门统计人数"
"先合并两个表，再去重"
```

## 五、常见问题

### Q: 系统不认识我的指令怎么办？

尝试改用更明确的说法：
- 错误：`"看看公司数据"` ← 太模糊
- 正确：`"筛选技术部的员工"` ← 有操作+条件

系统支持规则降级（无需 AI），但完全无规律的模糊表达可能无法理解。

### Q: 为什么我看到了 _lkp_ 开头的列？

这是 Match（匹配）操作的结果。`_lkp_` 前缀表示这些列来自被匹配的右表，例如 `_lkp_城市` 表示从另一张表匹配到的城市数据。

### Q: 如何撤销操作？

在左侧面板的"版本时间线"中点击上一个版本即可回退。系统最多保留 20 个版本快照。

### Q: 数据会泄露吗？

**不会。** 所有数据在本地浏览器中处理。AI 解析仅上传列名和示例值，不上传原始数据行。

### Q: 支持多大的文件？

建议单表不超过 30000 行或 10 MB。更大的文件可能会导致解析缓慢。

### Q: 没有 API Key 能用吗？

能。系统使用规则降级解析简单指令（单步操作），但复杂指令（流水线、公式）需要 AI 理解。
