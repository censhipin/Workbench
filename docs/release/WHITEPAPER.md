## 一、产品介绍

**表格数据工作台**是一款面向中国办公场景的**自然语言驱动数据处理桌面工具**。

用户上传 Excel 文件后，输入中文指令即可完成数据筛选、清洗、计算、聚合、匹配等操作。AI 理解用户意图，自动执行数据处理，并以可视化方式展示执行过程和结果。

> "像跟同事说话一样操作数据。"

产品形态为 **Electron 桌面应用 + Next.js Web 渲染**，数据全部在本地计算，不上传用户文件。

## 二、解决的问题

### 传统 Excel 操作的痛点

| 问题 | 表现 | 本产品的方案 |
|------|------|-------------|
| **公式门槛高** | 需要记住 VLOOKUP, IF, SUMIF 等数十个函数和参数 | 中文输入"按部门统计工资"即可 |
| **操作不可追溯** | 改错了 Ctrl+Z 只能回退一步 | 每次操作生成版本快照，可随时回退 |
| **多步操作复杂** | 筛选→公式→排序需要手动协调 | Pipeline 自动串联多步操作 |
| **数据质量难保证** | 空值、重复、格式不统一全靠肉眼检查 | 自动 Profile + Repair |
| **中文列名支持差** | 公式和 VBA 只认字母列号 | 天然支持中文列名 |

### 目标用户

- **业务人员**：需要分析数据但不会写公式/VBA
- **数据分析师**：需要快速验证想法、做临时处理
- **行政/财务/人事**：日常处理报表、花名册、工资表

## 三、核心能力

### 支持的 11 种数据操作

| 操作 | 能力 | 示例指令 |
|------|------|---------|
| **筛选 (Filter)** | 按条件过滤行，支持 AND/OR 组合 | "筛选技术部工资大于10000的" |
| **排序 (Sort)** | 单列或多列升序/降序 | "按年龄从大到小排序" |
| **聚合 (Aggregate)** | SUM/AVG/COUNT/MAX/MIN，支持分组 | "按部门计算平均工资" |
| **公式 (Formula)** | 四则运算，支持中文列名引用 | "收入 = 基本工资 * 绩效" |
| **更新 (Update)** | 批量替换列值，支持条件更新 | "将部门全部改成技术部" |
| **清洗 (Clean)** | 删除空行、去空格、格式化 | "删除空值数据" |
| **去重 (Dedup)** | 按列去重，保留首次出现 | "删除重复的订单ID" |
| **匹配 (Match)** | Left Join 关联两张表 | "把城市信息匹配到客户表" |
| **合并 (Merge)** | 纵向拼接多个表 | "合并三个月的销售数据" |
| **投影 (Projection)** | 保留/删除/重命名指定列 | "只看姓名和部门" |
| **流水线 (Pipeline)** | 多步操作串联执行 | "先筛选再排序最后计算" |

## 五、AI 执行流程

### 完整链路

```
用户： "帮我计算每个部门平均工资"
                        │
                        ▼
    ① NLU (DeepSeek / 规则降级)
       └─→ 识别：操作=sum, 聚合=AVG, 分组=部门, 目标=基本工资
                        │
                        ▼
    ② 编译 (TaskCompiler)
       └─→ columnHint → columnKey 绑定
       └─→ 生成 ExecutionPlan { type: 'aggregate', method: 'AVG', ... }
                        │
                        ▼
    ③ 数据画像 (DataProfile)
       └─→ 分析每列类型、空值率、唯一值
       └─→ 确认字段可用于聚合
                        │
                        ▼
    ④ 自动修复 (Repair)
       └─→ 必要时修正列引用/类型/空值
                        │
                        ▼
    ⑤ 计划验证 (Validate)
       └─→ 检查 plan 合法性
                        │
                        ▼
    ⑥ 执行 (V2 Executor)
       └─→ AggregateExecutor 分组计算平均工资
                        │
                        ▼
    ⑦ 验证 (Verification)
       └─→ 9 个 Verifier 逐条检查结果正确性
                        │
                        ▼
    ⑧ 解释 (Explain)
       └─→ 生成人类可读的执行说明
                        │
                        ▼
    结果展示 (UI)
       └─→ 表格 + 版本快照 + 执行计划 + 解释
```

### 关键设计点

- **AI First**：自然语言理解优先走 DeepSeek API，规则解析为降级方案
- **本地计算**：所有数据处理在客户端完成，不上传用户文件
- **版本化**：每次操作生成不可变数据快照，最多 20 个版本
- **插件化执行器**：新增操作类型只需注册一个 Executor

## 六、数据安全设计

### 安全原则

| 方面 | 设计 |
|------|------|
| **数据不离开本地** | 所有数据执行在客户端内存中完成 |
| **文件不上传** | 上传解析后仅在浏览器进程内处理 |
| **AI 只传 Schema** | DeepSeek API 仅接收列名/类型/示例值，不传原始数据 |
| **API Key 本地存储** | 使用浏览器 localStorage，不经过后端 |
| **IndexedDB 持久化** | 版本快照存储在客户端，不清除 |

### 数据流示意

```
用户 Excel → 浏览器 → SheetJS 解析 → 内存 DataSet
                                        │
                          ┌─────────────┴─────────────┐
                          │                           │
                    AI 理解 (传给 API)          数据处理 (本机执行)
                     仅列名/类型/示例值         不涉及网络传输
                          │                           │
                          └─────────────┬─────────────┘
                                        ▼
                                  结果展示
                            (仅存在于浏览器内存)
```

### 版本存储

- 每次操作保存完整数据快照（columns + rows）
- 上限 20 个历史版本
- 支持回退到任意历史版本
- 存储方式：浏览器 IndexedDB
