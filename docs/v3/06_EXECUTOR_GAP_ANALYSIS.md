# 11 个 Executor 逐个缺口分析

## 分析模板

每个 Executor 分析以下维度：
- **当前行为**：做什么、怎么做的
- **V2 缺口**：当前缺少什么能力
- **EIC Profile 价值**：数据画像能提前发现什么
- **EIC Validate 价值**：智能校验能预防什么
- **EIC Explain 价值**：执行解释能补充什么
- **自动修复机会**：EIC Repair 能自动处理什么

---

## 1. FilterExecutor

**当前行为**：接收条件数组 → 调用 `filterRowsMulti()` → 返回过滤后的行。

**V2 缺口**：
- 无条件值预检查：不知道 "销售额 > 10000" 这个值在数据中是否存在
- 无结果预估：不知道会过滤掉多少行
- 0 匹配行无解释：用户不知道"为什么一行都没匹配"
- 列不存在直接抛错

**EIC Profile**：执行前检查条件列的值分布、空值率、唯一值率；检查条件值是否存在于数据中。

**EIC Validate**：预估过滤后的行数，如果结果为空则警告；检查多条件组合是否合理。

**EIC Explain**："筛选条件"销售额 > 10000 匹配了 3/100 行（3%），因为大部分行销售额低于 10000"。

**自动修复**：列不存在 → 模糊匹配候选列；条件值不存在于数据中 → 建议相似值。

---

## 2. SortExecutor

**当前行为**：接收排序列和方向 → 调用 `sortRowsMulti()` → 返回排序后的行。

**V2 缺口**：
- 无类型感知排序：文本列和数字列排序行为不同，但无预检
- 混合类型列排序可能产生意外结果
- 排序对大数据量的性能影响无预估

**EIC Profile**：检查排序列的类型分布、空值率。

**EIC Validate**：如果排序列高空值率，警告空值行会排在开头/末尾。

**EIC Explain**："按'销售额'降序排列，3 行空值排在末尾"。

**自动修复**：空值排序位置策略可配置。

---

## 3. AggregateExecutor

**当前行为**：接收聚合列、方法和 groupBy → 调用 `aggregateRows()` → 返回聚合结果。

**V2 缺口**：
- **非数值列求和/平均 → NaN，无警告**
- groupBy 列的唯一值太多 → 结果行数爆炸，无预警
- 空值处理策略不透明
- 多列聚合的列合并依赖长度匹配，不匹配则静默错误

**EIC Profile**：检查聚合列是否数值类型；检查 groupBy 列的唯一值计数；检查空值比例。

**EIC Validate**：如果聚合列非数值 → 警告并推荐数值列；如果 groupBy 唯一值 > 50 → 提示结果会很大。

**EIC Explain**："对'销售额'平均，结果 12,345.67。警告：3 行包含非数值数据已跳过"。

**自动修复**：非数值行自动跳过（带标注）；非数值列自动推荐候选数值列。

---

## 4. DedupExecutor

**当前行为**：接收列名 → 调用 `dedupRows()` → 返回去重后的行。

**V2 缺口**：
- 无影响预估：用户不知道会删掉多少行
- 无意中的数据丢失无预警
- 无去重键的建议

**EIC Profile**：按去重列检查重复率；检查去重列的空值率（空值行可能被误去重）。

**EIC Validate**：如果去重将删除 > 30% 的行 → 高亮警告；如果按空值率高的列去重 → 提示风险。

**EIC Explain**："按'姓名'去重后保留 55 行，删除了 45 个重复项。其中 3 行姓名为空，已保留首次出现的空值行"。

**自动修复**：不可用（数据修改需要用户确认）。

---

## 5. MatchExecutor

**当前行为**：接收匹配键和查找表 → 调用 `matchMultiTables()` → 返回匹配结果。

**V2 缺口**：
- 无主键质量检查：如果匹配列不是唯一键，可能产生意料之外的笛卡尔积
- 无匹配率预估：不知道多少行会匹配上
- 模糊匹配阈值固定（Levenshtein 0.85），不可配置
- 多匹配键时无主键去重警告

**EIC Profile**：检查匹配列在两张表中的唯一值率和重复率；预估匹配率。

**EIC Validate**：如果匹配列重复率高 → 警告可能产生多对多匹配；如果预估匹配率 < 50% → 提示。

**EIC Explain**："表 A 的 100 行中，85 行在表 B 找到了匹配（85% 匹配率）。15 行未匹配（已追加在末尾）"。

**自动修复**：列名匹配不上时自动模糊查找；主键不唯一时可自动去重后匹配。

---

## 6. MergeExecutor

**当前行为**：接收源表名 → 调用 `mergeTables()` → 返回纵向合并结果。

**V2 缺口**：
- 无列结构检查：两张表列结构不同时，合并可能产生大量空值
- 无行数预估

**EIC Profile**：比较两张表的列结构（共有列、独有列）；检查列类型兼容性。

**EIC Validate**：如果列结构差异大 → 警告合并后会有大量空值。

**EIC Explain**："表 A（5 列 x 100 行）与表 B（5 列 x 200 行）合并。表 A 独有列：'备注'，表 B 独有列：'原因'，不匹配的列值为空"。

**自动修复**：列结构不匹配时建议对齐后再合并。

---

## 7. CleanExecutor

**当前行为**：接收目标列（或全部列）→ 调用 `cleanData()` → 清理空行和异常值。

**V2 缺口**：
- 无清洗影响预估
- 无清洗内容详细报告
- 无清洗策略建议

**EIC Profile**：检查空行比例、空值分布、异常值模式。

**EIC Validate**：如果空行比例过高 → 确认是否真的需要清洗。

**EIC Explain**："已清洗 10 行空数据，8 个异常值。清洗后剩余 90 行"。

**自动修复**：可配置清洗策略（保守/激进）。

---

## 8. ProjectionExecutor

**当前行为**：接收 include/exclude/rename/reorder → 操作行列。

**V2 缺口**：
- 列不存在静默跳过
- 重命名冲突无警告
- 排除所有列不检查

**EIC Profile**：检查列存在性；检查重命名后的重复列名。

**EIC Validate**：如果排除包含某列但后续操作引用该列 → 警告。

**EIC Explain**："已保留 5 列：'姓名'、'部门'、'销售额'... 排除了 2 列：'备注'、'内部ID'"。

**自动修复**：列不存在 → 模糊匹配建议。

---

## 9. UpdateExecutor

**当前行为**：接收目标列、值、条件 → 批量修改。

**V2 缺口**：
- 无影响行数预估
- 值类型与列类型不匹配无检查
- 无条件更新无警告

**EIC Profile**：检查值类型与列类型是否匹配；预估满足条件的行数。

**EIC Validate**：如果更新会修改所有行 → 确认；如果条件值不存在 → 警告。

**EIC Explain**："将'部门'= '技术部'的行的'级别'更新为'高级'，影响了 15/100 行"。

**自动修复**：类型不匹配时自动转换。

---

## 10. FormulaExecutor

**当前行为**：接收表达式参数 → 逐个单元格计算 → 返回结果。

**V2 缺口**：
- **NaN 静默转 0，无提示**
- **除零返回 Infinity，无提示**
- 日期解析格式覆盖不全
- 条件聚合（SUMIF/COUNTIF）无空值策略说明

**EIC Profile**：检查 sourceColumns 中的数字/空值分布；检查目标列是否存在（决定覆盖还是新增）。

**EIC Validate**：如果 sourceColumns 有空值 → 提示空值处理方式；如果目标列已存在 → 确认覆盖。

**EIC Explain**："计算'总价 = 数量 × 单价'，新增列'总价'，100/100 行计算成功。2 行因单价为空计算结果为 0"。

**自动修复**：空值单元格自动视为 0（带标注）；日期格式自动尝试多种解析。

---

## 11. PipelineExecutor

**当前行为**：接收步骤数组 → 逐个执行，上步输出为下步输入。

**V2 缺口**：
- 中间步骤失败 → 整个 pipeline 终止，无部分结果
- 步骤间列结构变更无追踪
- 无步骤级性能监控
- 无 checkpoint/rollback

**EIC Profile**：预分析所有步骤的输入输出列变化。

**EIC Validate**：检查步骤是否可衔接（上步的输出列是否满足下步的输入要求）。

**EIC Explain**："13 步流水线执行完成。第 5 步（筛选）减少了 40 行，第 8 步（公式）新增了 2 列"。

**自动修复**：单步失败时可跳过并继续（带标注）。
