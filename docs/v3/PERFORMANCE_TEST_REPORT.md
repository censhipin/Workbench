## 六、全链路耗时预估（30000行）

```
操作                 耗时
──────────────────────────────
选择文件 + 上传       ~3,000ms  ← XLSX 解析占大头
AI 解析 (DeepSeek)    ~1,500ms  ← API 调用网络延迟为主
执行 (Pipeline 3步)   ~750ms
导出                  ~3,000ms
──────────────────────────────
合计                  ~8,250ms
```

> 注：AI 解析耗时取决于 DeepSeek API 响应时间（1-3s），  
> 非本地计算瓶颈。执行和导出均在本地，瓶颈在 SheetJS 的写入。

## 七、结论

| 结论 | 状态 |
|------|------|
| **30000行 正常运行** | ✅ **通过** |
| Filter 30000行 < 300ms | ✅ 212ms |
| Formula 30000行 < 500ms | ✅ 337ms |
| Aggregate 30000行 < 300ms | ✅ 176ms |
| Pipeline 3步 30000行 < 1000ms | ✅ 738ms |
| 导出 30000行 < 5000ms | ✅ 3,093ms |
| 所有操作 O(n) 线性扩展 | ✅ |

### 性能瓶颈排序

1. **XLSX 解析**（30000行 ~2.6s）— SheetJS `read` 最重
2. **XLSX 导出**（30000行 ~3.0s）— SheetJS `write` 同样重
3. **JSON 深拷贝**（30000行 ~140ms）— 已优化为 structuredClone
4. **执行引擎**（30000行 Pipeline ~750ms）— 纯内存操作，瓶颈不在 CPU

### 优化建议（如果后续需要）

| 问题 | 建议 |
|------|------|
| XLSX 解析慢 | 使用 `exceljs` stream 模式或 WASM 加速的 SheetJS Pro |
| 导出慢 | 前端导出可为 30000行 + 显示分页进度 |
| 版本快照过大 | 差量存储或仅存操作日志 (CRDT) |
| JSON 内存 | 30000行约 60MB 堆 + 3.5MB 序列化，在浏览器中可接受 |
