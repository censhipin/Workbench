## 已知问题

### 1. V2 AggregateVerifier 浮点数精度问题

**位置**：`lib/v2/verifier/AggregateVerifier.ts:99`

**问题**：使用 `Number(actual) !== Number(expected)` 做严格相等比较，无法容忍公式计算后 AVG 聚合产生的浮点精度误差（如 `36660.67` vs `36660.6700766284`）。

**影响**：仅影响带 Formula 步骤后的 AVG 聚合测试。纯数据的 AVG 聚合（如员工基本工资）不受影响。

**建议修复**：将比较改为 `Math.round(actual * 100) / 100 === Math.round(expected * 100) / 100` 或使用容差比较。
